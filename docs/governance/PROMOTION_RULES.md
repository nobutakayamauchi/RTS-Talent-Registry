# PROMOTION_RULES

## Promotion Levels
1. `starred`
2. `candidate`
3. `trial`
4. `bench`
5. `specialist`
6. `core_staff`
7. `retired`

## Baseline gates
- `starred -> candidate`: intake record exists, source reference exists.
- `candidate -> trial`: initial scorecard completed, security checklist started.
- `trial -> bench`: route fit identified, rollback/retirement path documented.
- `bench -> specialist`: repeated successful evaluations with low-risk profile.
- `specialist -> core_staff`: sustained reliability + governance approval.
- `* -> retired`: unsupported, risky, stale, or superseded.

## Required evidence blocks
Each promotion decision should include:
- `confirmed_facts`
- `assumptions`
- `unverified`
- `risks`
